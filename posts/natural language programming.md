I was never a very good coder, I came to terms with that some time ago.
But times they are a changing. It now appears possible to program exclusively using natural language.

There’s lots of debate about just how “possible” this is, mostly around whether it actually makes sense to do this kind of coding in production. I feel like the answer is Yes, with a lot of asterisks. But that’s not really what I care about right now.

What I’m interested in is creating my own personal computing environment, user-base of one, me.


Programming is the act of specifying behavior.
A program is a behavior that is defined. The language is precise.
English has a lower definition level, more ambiguity, but more flexibility.
Natural language is a programming language of its own, it just requires an interpreter (language model), just as classical coding languages require a step that translates them into machine code.

The flexibility affords power and speed that more rigid language does not, but it’s not without its drawbacks.
You risk underspecification at all times.
It’s a crutch to rely on the language model itself to do the semantic lifting for you, you need to be disciplined about what you’re actually saying.
“This isn’t working."
“Something’s wrong here, fix this”.
A model can go in circles forever when prompted in this way.
To effectively program with natural language, you still have to command the language.

I don’t like the slot machine metaphor.
It implies the output is extremely random, when in fact it is not. It’s not purely deterministic, but it’s not random at all.
When trying to program with NL, saying that the whole paradigm is a slot machine denies that there are any real decisions being made by the programmer, the prompter.

I/O with models is dialectic. Every possible output exists latently within the weights of the model.
